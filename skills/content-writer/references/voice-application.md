# Voice Application

วิธี translate `profile.voice` ไปยัง choices ตอนเขียน

## tone

| profile.tone | คำที่ใช้ | ตัวอย่างประโยค |
|--------------|----------|-----------------|
| กันเอง       | "เรา", "ใช่มั้ย", "ลองดู" | "ลองนี่ดูเลย เพื่อนเราใช้แล้วชอบ" |
| ทางการ       | "ดิฉัน/ผม", "ขอแนะนำ", "เรียน" | "ทีมเราขอแบ่งปันข้อมูลที่ค้นพบ..." |
| ขายของ       | "เด็ดมาก", "ห้ามพลาด", "โปรพิเศษ" | "แค่วันนี้! โปรลด 50% ห้ามพลาดเด็ดขาด" |
| วิชาการ      | "งานวิจัยพบว่า", "สถิติ", "หลักการ" | "งานวิจัยพบว่ากว่า 60% ของผู้ใช้..." (ต้องมี source!) |
| ตลก          | quote pop culture, exaggeration | "นั่งวิเคราะห์ครีมเหมือนนักวิทย์ NASA" |

## perspective

ตรวจให้สม่ำเสมอตลอดบทความ — สลับกลางทางทำให้ disorienting

| perspective | ใช้แทน | ตัวอย่าง |
|-------------|--------|----------|
| ผม          | "ผม", "ของผม", "ผมเลย" | "ผมเริ่มสังเกตว่า..." |
| ดิฉัน        | "ดิฉัน", "ของดิฉัน" | "ดิฉันเลยลองทดสอบดู..." |
| เรา         | "เรา", "ของเรา", "ทีมเรา" | "เราสังเกตว่าลูกค้ามักถาม..." |
| ทีม <แบรนด์> | "ทีม [name]", "พวกเรา" | "ทีม Dermalux ขอเล่าให้ฟังนะคะ..." |

## formality_level

ปรับ:
- คำลงท้าย: casual ใช้ "นะ"; semi-formal ใช้ "ครับ/ค่ะ" บางที่; formal ใช้ "ครับ/ค่ะ" เกือบทุกย่อหน้า
- คำเรียกผู้อ่าน: casual = "เพื่อนๆ"; semi = "ทุกคน"; formal = "ท่าน"
- คำเชื่อม: casual = "แบบว่า"; formal = "ในส่วนของ", "ทั้งนี้"

## emoji

ทำตาม profile.voice.use_emojis + emoji_style:

- **minimal** (use_emojis: true): 1-2 ตัวต่อโพสต์ ตำแหน่ง hook + CTA
- **expressive**: 4-8 ตัวกระจายทุก 2-3 ประโยค
- **off** (use_emojis: false): 0 ตัว — ไม่ใส่แม้เป็น CTA

## sentence_length

| setting | guideline |
|---------|-----------|
| สั้น    | เฉลี่ย 8-12 คำ/ประโยค, ห้ามเกิน 18 |
| กลาง    | 12-18 คำ, ผสม long/short |
| ผสม     | กระจายมาก — บางประโยค 3 คำ บางประโยค 25 คำ — สร้างจังหวะ |

## signature_phrases

ทุกบทความใส่อย่างน้อย 1 ครั้ง — แต่ห้าม forced
- ดี: ใส่ตอน transition ระหว่างย่อหน้า
- ไม่ดี: ใส่ที่กลาง paragraph แล้วทำให้ flow แตก

ถ้ามี signature_phrases หลายตัว — สลับเลือก ไม่ใช้ตัวเดียวซ้ำในบทความเดียว

## banned_words

เช็ค output ทุกครั้งก่อน finalize ใช้ exact-match search

ถ้าเจอ:
1. หา synonym ที่ใกล้ที่สุด
2. ถ้าหาไม่ได้ — เปลี่ยนโครงประโยค
3. ถ้าจำเป็นต้องใช้คำใกล้เคียง (เช่น banned: "รับรอง" แต่ context ต้องการคำคล้าย) — ใช้ "เห็นผลจริง", "ทดสอบแล้ว" แทน
