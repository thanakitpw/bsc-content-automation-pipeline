# Humanize Writing — เขียนให้เหมือนคน ไม่ใช่ AI

★ ★ ★ **content-writer ต้องอ่านไฟล์นี้ทุกครั้งก่อนเขียน + ก่อน Self-Check ทุกครั้ง** ★ ★ ★

ไฟล์นี้รวบรวม rules ที่บังคับใช้กับทุกเพจ ทุก type — เป็น default behavior ถ้าเพจไหนต้องการ override (เช่น เพจวิชาการที่อยาก formal จริงๆ) ให้บันทึกใน `style-notes.md` ของเพจนั้น

---

## Big Picture: ทำไม AI writing sound เหมือน AI

จาก research:

1. **Sentence uniformity** — ทุกประโยคยาวเท่าๆ กัน, structure ซ้ำกัน → ไม่มี rhythm
2. **Translation-style โครงสร้าง** — ภาษาไทยจาก AI มักโครงสร้างเหมือน Google Translate
3. **Over-explaining + filler** — อธิบายซ้ำ, ใส่คำเชื่อมเกินจำเป็น
4. **Formal tone ผิดที่** — ใช้คำเขียนที่คนพูดจริงไม่ค่อยใช้
5. **Lack of lived experience** — ไม่มี "เคย", "ผมเจอ", "เห็นกับตา"
6. **Predictable vocabulary** — ใช้คำกลางๆ ที่คาดเดาได้

---

## 🚫 Hard Rules (บังคับใช้ทุกครั้ง)

### Rule 1: ห้ามอ้างอิงงานวิจัย / สถิติทางการ
> "งานวิจัยพบว่า..." / "ตามสถิติ..." / "การศึกษาในปี XXXX"

ทำให้ฟังจริงจัง ไกลตัว เหมือน text book ไม่ใช่คนเล่าเรื่อง

**แทนที่ด้วย**:
- "ที่ผม/เราเจอ..."
- "เพื่อนเคยเล่าว่า..."
- "มีคนเคยลอง..."
- "ลองสังเกตดู..."

ถ้า research file มีข้อมูลที่อ้างอิง — เปลี่ยนเป็น narrative ที่ specific แต่ไม่ academic เช่น "เพิ่งอ่านเจอว่า..." แทน "งานวิจัยปี 2024 ระบุว่า..."

### Rule 2: ใช้คำเข้าใจง่าย ไม่ใช้ศัพท์ทางวิชาการ/อังกฤษ ที่ไม่จำเป็น

❌ "stress level สูง"
✅ "เครียดมาก" / "ความเครียดสะสม"

❌ "มี anxiety จาก work-life balance ที่ไม่ดี"
✅ "เครียดเพราะงานล้นจนไม่มีเวลาพัก"

❌ "optimize productivity"
✅ "ทำงานให้ได้มากขึ้น" / "ทำงานเร็วขึ้น"

❌ "implement strategy"
✅ "ลองทำตามแผนนี้" / "เอาไปใช้"

**หลักการ**: ใช้ภาษาที่คุณป้าข้างบ้านอ่านแล้วเข้าใจ ถ้าศัพท์ tech/อังกฤษไม่จำเป็น ตัดทิ้ง ถ้าจำเป็นจริงๆ (เช่น brand name, technical term ที่แทนยาก) ให้คงไว้แต่อธิบายเสริม

### Rule 3: Date/Time format

❌ "Day 1-3", "Day 1: ทำ X"
✅ "วันที่ 1-3", "วันแรก: ทำ X"

❌ "Week 1", "Month 2"
✅ "อาทิตย์แรก", "เดือนที่ 2"

❌ "ปี 2018 มีงานวิจัย..."
✅ "เมื่อ 7-8 ปีก่อน มีคนเคย..."
✅ "เมื่อหลายปีที่แล้ว..."
✅ "ปีที่แล้วมีคน..."

❌ "เดือน October 2025"
✅ "ตุลาคมปีที่แล้ว"

**หลักการ**: ใช้เวลาแบบสัมพัทธ์ (relative) มากกว่าตัวเลขเป๊ะ — คนพูดกัน "เมื่อปีที่แล้ว" ไม่ใช่ "ในปี 2025"

### Rule 4: ใช้คำที่เป็นภาษาพูด ไม่ใช่ภาษาเขียนเป็นทางการ

| ทางการ (หลีกเลี่ยง) | กันเอง (ใช้แทน) |
|---|---|
| สัปดาห์ | อาทิตย์ |
| ในขณะที่ | ตอนที่ |
| ดังนั้น | เลย, ก็เลย |
| อย่างไรก็ตาม | แต่ |
| นอกจากนี้ | แล้วก็, อีกอย่าง |
| เนื่องจาก | เพราะ |
| ด้วยเหตุนี้ | เลย |
| สำหรับ | สำหรับ → ใช้ "สำหรับ" ได้ แต่บางที่ตัดออกได้เช่น "ดีสำหรับคนที่..." → "ดีกับคนที่..." |
| ข้าพเจ้า | ผม / ดิฉัน / เรา |
| ท่าน | คุณ |
| มากมาย | เยอะ |
| ปัจจุบัน | ตอนนี้ |
| ในอนาคต | ต่อไป |

**ยกเว้น**: เพจ type=`personal_brand` ที่ profile.voice.formality_level=formal — ให้ใช้ทางการ

### Rule 5: หลีกเลี่ยง "AI tic words" — คำที่ AI ชอบใช้แต่คนไทยจริงไม่พูด

❌ "ในยุคปัจจุบัน...", "ในโลกที่เปลี่ยนแปลงอย่างรวดเร็ว..."
❌ "อย่าลืมว่า...", "สิ่งสำคัญคือ..."
❌ "ในที่สุด...", "สรุปแล้ว..."
❌ "ทั้งนี้และทั้งนั้น..."
❌ "อย่างมาก", "อย่างยิ่ง", "เป็นอย่างยิ่ง"

ถ้าจะเปิดบทความ — เปิดด้วย scene, คำถาม, หรือ observation ตรงๆ ไม่ใช่ "ในยุคที่..."

### Rule 6: หัวเราะใช้ "555" ไม่ใช่ "ฮ่าฮ่าฮ่า"

★ Thai-specific — คนไทยในแชท/social media ใช้ตัวเลขแสดงเสียงหัวเราะ

❌ "ฮ่าฮ่าฮ่า" / "ฮ่ะฮ่ะ" / "5555555" ยาวเกินไป
✅ "555" (เบสิก) / "55+" / "5555" (ขำมาก)

**ใช้เมื่อ**:
- โพสต์ tone กันเอง / ตลก / casual
- ตอนเล่าเรื่องตัวเองที่ตลก / ผิดพลาด
- comment style

**หลีกเลี่ยงเมื่อ**:
- เพจ tone formal / serious
- ถ้า `profile.voice.tone` = "ทางการ" / "วิชาการ" / "expert" → ห้ามใช้
- หลังประโยคเศร้า / ซีเรียส (ดูแปลก)
- ใน hook (ใช้ในเนื้อหาได้ — ไม่ควรเปิดบทความ)

**Variation**:

| Form | ความรู้สึก |
|------|-----------|
| `555` | ขำเบาๆ |
| `55+` / `555+` | ขำมารยาท / สุภาพ |
| `5555` | ขำเอาจริง |
| `55555` | ขำหนัก / ตลกมาก |
| `ฮา` | สั้นกว่า 555 — บางคนใช้ |

**ตัวอย่าง**:

❌ "ผมทำพังเองครับ ฮ่าฮ่าฮ่า"
✅ "ผมทำพังเองครับ 555"

❌ "เพื่อนเล่ามา ฮ่ะฮ่ะ ขำหนักมาก"
✅ "เพื่อนเล่ามา 5555 ขำหนัก"

❌ "ลืมเครดิตการ์ด ฮ่าๆ ต้องวิ่งกลับบ้าน"
✅ "ลืมเครดิตการ์ด 55 ต้องวิ่งกลับบ้าน"

## 🎯 Burstiness — ผสมประโยคสั้น/ยาว

★ key technique ที่ทำให้ฟังเป็นคน

AI เขียนประโยคยาวสม่ำเสมอ คนเขียนผสม:

ตัวอย่างที่ดี (มี rhythm):
> "เครียดมาก. เครียดจนนอนไม่หลับ ตื่นมาตี 3 มาตี 4 ทุกวัน — ลองทำตามที่หมอบอกหมดแล้ว ก็ยังไม่ดีขึ้น"

ตัวอย่างที่ไม่ดี (uniform):
> "ความเครียดส่งผลต่อการนอนของคุณ ทำให้คุณตื่นกลางดึก คุณควรพยายามแก้ไขปัญหานี้โดยปฏิบัติตามคำแนะนำของแพทย์"

**กฎคร่าวๆ**: ทุก 2-3 ประโยคยาว ใส่ประโยคสั้น (3-5 คำ) คั่น 1 ตัว

## 👤 Embed Lived Experience (Cyborg approach)

★ ทำให้ content "ฟังเป็นคน" ที่สุด

จาก research: คนเขียน + AI ช่วย amplify > AI เขียนทั้งหมด

**Technique**: ทุกบทความใส่ "anchor of experience" อย่างน้อย 1 จุด:

- "เมื่อ 2-3 อาทิตย์ก่อนเจอลูกค้าคนหนึ่ง..." (case)
- "ผมเองก็เคยเป็น..." (personal)
- "เพื่อนคนหนึ่งบอกว่า..." (anecdote)
- "ลองทำดูเอง — ที่เห็นผลคือ..." (experiment)

ถ้าไม่มี real story ใน profile/research:
- ดึงจาก research.reference_quotes — quote จริงจาก Pantip/Reddit เป็น "anchor"
- ถ้าไม่มี quote — เปลี่ยน hook เป็น "สังเกตได้ไหมว่า..." แทน "ในยุคปัจจุบัน..."
- ★ creator type: ถามผู้ใช้ "มีเคสจริงที่คุณเคยเจอเรื่องนี้ไหม?" ก่อนเขียน — story จากผู้ใช้จริงดีที่สุด

**ห้ามแต่งเรื่อง** — ถ้าไม่มี real anchor ก็ใช้ generalization ที่ honest:
- ✅ "หลายคนที่ผมคุยด้วยบอกว่า..."
- ❌ "เพื่อนสนิทผมชื่อสมศรีเล่าว่า..." (แต่งเรื่อง)

## 🔊 Read-Aloud Test

ก่อน output — อ่านออกเสียงในใจ ถ้าเจอประโยคที่:

- พูดแล้วยากเข้าใจในรอบเดียว
- ต้องหยุดคิดกลางประโยค
- ฟังแปร่งหู เหมือนตำราเรียน

→ rewrite ให้สั้นลงหรือแบ่งเป็น 2 ประโยค

## ✂️ Cut Filler Words

AI ชอบใส่ filler — ตัดทิ้ง:

| Filler | Replacement |
|---|---|
| "เป็นที่ทราบกันดีว่า..." | (ตัดทิ้ง) |
| "อย่างที่หลายคนรู้กัน..." | (ตัดทิ้ง) |
| "ในความเป็นจริงแล้ว..." | "จริงๆ แล้ว..." หรือตัด |
| "อีกประการหนึ่ง..." | "อีกอย่าง..." |
| "ในแง่ของ..." | "เรื่อง..." |
| "ซึ่ง" | (ตัดออกหลายที่ได้ — ภาษาพูดไม่ค่อยใช้) |

**Rule**: ถ้าตัดประโยค/วลีออกแล้วความหมายไม่เปลี่ยน = ตัดทิ้ง

## ✅ Self-Check Checklist (ก่อน output ทุกครั้ง)

content-writer ต้องผ่าน 9 ข้อนี้:

- [ ] **R1** ไม่มีคำว่า "งานวิจัย", "การศึกษา", "สถิติ" + ปีอ้างอิง
- [ ] **R2** ไม่มีศัพท์อังกฤษทาง psychology/business ที่ไม่จำเป็น (stress level, anxiety, productivity, optimize, implement, KPI ที่ไม่ใช่ profile keyword)
- [ ] **R3** ไม่มี "Day X" / "Week X" / "Month X" — ใช้ "วันที่ X" / "อาทิตย์" / "เดือน" แทน
- [ ] **R3.1** ไม่มีปีตัวเลขเป๊ะ (เช่น 2018, 2024) ใช้ "ปีที่แล้ว", "หลายปีก่อน", "เมื่อ X ปีก่อน"
- [ ] **R4** ไม่มี "สัปดาห์" → ใช้ "อาทิตย์" (ยกเว้น formality_level=formal)
- [ ] **R5** ไม่มี AI tic words: "ในยุคปัจจุบัน", "อย่างมาก/ยิ่ง", "ทั้งนี้และทั้งนั้น", "ในขณะที่", "ดังนั้น" (ใช้ "เลย" แทน)
- [ ] **R6** ถ้ามีหัวเราะ ต้องเป็น "555" / "5555" / "55+" — ห้าม "ฮ่าฮ่าฮ่า", "ฮ่ะฮ่ะ" (ยกเว้น tone formal/วิชาการ — ห้ามใส่หัวเราะเลย)
- [ ] **B** มี burstiness — ประโยคยาว/สั้นผสมกัน, มีประโยคสั้น (≤5 คำ) อย่างน้อย 2 ตัว
- [ ] **L** มี lived experience anchor อย่างน้อย 1 จุด — case / personal / anecdote / experiment

ถ้า fail ข้อใด ★ **ต้อง revise ก่อน output** ★

## Override Mechanism

ถ้าเพจไหนอยากใช้คำทางการ (เช่น เพจกฎหมาย, การแพทย์, วิชาการเข้มข้น) ให้บันทึกใน `style-notes.md` ของเพจ:

```markdown
## YYYY-MM-DD - Override

**Rule**: เพจนี้ override humanize-writing rules — ใช้ "สัปดาห์" แทน "อาทิตย์", ใช้ "ดังนั้น" ได้, อนุญาตอ้างอิงงานวิจัยแบบมี citation

**Rationale**: เพจสาย <field> — formal tone จำเป็นเพื่อ credibility
```

content-writer จะอ่าน style-notes ก่อน humanize-writing — overrides มาก่อน
