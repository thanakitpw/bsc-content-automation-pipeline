# Post Formatter

## Purpose

แปลง draft ที่ user แก้เสร็จแล้ว → final post ที่:
- พร้อม **copy paste** ลงเพจทันที (FB, IG, อื่นๆ)
- อยู่ใน **folder ต่อโพสต์** — รวม post + cover image (ถ้ามี) ไว้ที่เดียว
- ตั้งชื่อด้วย **เลขลำดับ** (1, 2, 3...) ต่อเพจ — เห็น progression ของแต่ละเพจชัด

## Folder Convention ★

```
<workspace>/
├── drafts/<slug>/
│   └── 2026-05-11-ai-misconception.md      ← raw (มี frontmatter)
└── final/<slug>/                             ← ★ output ของ skill นี้
    ├── 1-ai-misconception/                   ← folder ต่อโพสต์
    │   ├── 1-ai-misconception.md             ← เนื้อหาพร้อม copy
    │   └── cover.png                         ← (user วางเอง ถ้ามี)
    ├── 2-prompt-tips/
    │   └── 2-prompt-tips.md
    └── 3-.../
        └── 3-....md
```

**Folder ต่อ post** — ทำให้ user วาง cover image / asset อื่นๆ ที่ folder เดียวกันได้ ไม่กระจาย

**Per-slug numbering** — แต่ละเพจมี sequence ของตัวเอง เริ่ม 1 ใหม่

## Preflight

1. **draft**: `drafts/<slug>/<file>.md`
   - ผู้ใช้ระบุไฟล์ → ใช้ตัวนั้น
   - ไม่ระบุ → ใช้ draft ใหม่สุดของ slug
   - ไม่มี draft → "ยังไม่มี draft รัน content-writer ก่อน" หยุด

2. **profile**: `business-profiles/<slug>/profile.yaml`
   - อ่าน `hashtags.always`, `hashtags.rotating`, `cta.links`
   - business_type (กำหนด tone ของ alt-text + format)

3. **ตรวจ folder `final/<slug>/`**:
   - ไม่มี → สร้าง
   - มีอยู่แล้ว → scan **sub-folders** (ไม่ใช่ไฟล์) หา max number ปัจจุบัน

## Auto-Numbering Logic

scan sub-folders ที่ขึ้นต้นด้วยตัวเลขใน `final/<slug>/`:

```bash
SLUG_DIR="final/<slug>"
mkdir -p "$SLUG_DIR"
MAX_N=$(ls -d "$SLUG_DIR"/*/ 2>/dev/null | xargs -n1 basename | grep -oE '^[0-9]+' | sort -n | tail -1)
NEXT_N=$((${MAX_N:-0} + 1))
```

## Topic Slug Generation

จาก draft frontmatter หา `hook_angle` หรือ topic title:

1. ถ้า draft มี `topic:` ใน frontmatter → ใช้ตัวนั้น
2. ไม่งั้น — ดึงจากชื่อไฟล์ draft: `2026-05-11-ai-misconception.md` → `ai-misconception`

แปลงเป็น kebab-case (ภาษาอังกฤษ recommend, ไทยก็ได้):
- ตัด stop words: "การ", "เรื่อง", "ที่"
- เก็บ keyword หลัก 3-5 คำ
- ใช้ `-` คั่น

ตัวอย่าง:
- `ai-misconception`
- `prompt-tips-for-marketer`
- `วิธี-เริ่มต้น-ai`

## Format Conversion

### 1. ตัด frontmatter ออก

draft มี:
```yaml
---
business: ai-edu
platform: facebook
date: 2026-05-11
hook_angle: misconception reveal
...
---
```

→ ตัดทิ้งทั้ง block

### 2. Process body

**Link routing** (FB):
- หา URL ทุกตัวใน body
- ตัดออก → เก็บไว้สำหรับ "first comment" section
- ถ้าใน body มีคำว่า "ลิงก์ใต้คอมเม้นต์" หรือ "link in bio" → คงไว้

**Hashtag arrangement**:
- ลบ hashtag ทั้งหมดจากกลางบทความ (ถ้ามี)
- รวบกับ `profile.hashtags.always` + 1-2 จาก rotating ที่ตรง topic
- วางท้ายโพสต์ (เว้น 1 บรรทัด)
- FB: ≤5 hashtags; IG: 8-15

**Line breaks**:
- ตรวจให้มี blank line ระหว่างย่อหน้า
- ห้ามมี trailing whitespace

### 3. Generate alt-text (ถ้ามี cover)

ถ้า `cover.png` มีอยู่ใน folder ของ post (user วางไว้ก่อน) หรือ `cover_image` ใน frontmatter draft → generate alt-text:
- จาก context ของ post (topic + visual context)
- ภาษาไทย ≤125 chars
- บอกสิ่งที่จะเห็นในรูป (ตามเนื้อหา) — ไม่ขึ้นต้น "ภาพของ"

ถ้าไม่มี cover → set alt-text = "—" (dash)

## Output Format

ไฟล์ `final/<slug>/<N>-<topic>/<N>-<topic>.md`:

```markdown
<post body — clean text พร้อม paste>

<hashtags>

---

📎 Cover image: <relative path "cover.png" ถ้ามี หรือ "—">
🔗 Link for first comment: <url or "—">
✏️ Alt-text: <auto-generated or "—">

📊 Char count: <X> | Hashtags: <N>/5 (FB)
```

**ส่วน metadata footer** ใต้ `---`:
- ไม่ใช่ส่วนของโพสต์ที่จะ copy ลงเพจ
- ใช้ดูสำหรับ user ตอน paste — รู้ว่าต้องใส่อะไรเป็น first comment, alt-text ของรูปคืออะไร

## Output Message ★ (สำคัญ — ห้ามใส่อะไรเกินนี้)

หลัง save ให้แสดงข้อความนี้ตรงๆ:

```
✅ Ready to paste

📁 Folder: final/<slug>/<N>-<topic>/
📄 File: final/<slug>/<N>-<topic>/<N>-<topic>.md

วิธีใช้:
  1. เปิดไฟล์ .md — copy ส่วนเหนือ "---" → paste ลงเพจ
  2. ส่วนใต้ "---" คือ metadata (link, alt-text)

📊 <X> chars · <N> hashtags

รูปประกอบ (optional):
  วาง cover.png ใน folder เดียวกันก่อนโพสต์ ถ้าต้องการรูป
```

### ห้ามใส่ในข้อความ output ★

- ❌ ห้ามแนะนำให้รัน script ใดๆ (`bash scripts/...`, `python ...`) — plugin **ไม่มี script ในระบบ**
- ❌ ห้ามอ้างอิงถึง `image-generator`, `fal.ai`, `nano-banana` — skill เหล่านี้ถูกลบไปแล้ว
- ❌ ห้ามแนะนำ AI image tool ใดๆ (Canva ระบุได้ในฐานะ option แต่ไม่ใช่ใน output ของ skill นี้)
- ❌ ห้ามใส่ "warning" หรือ "note" เกี่ยวกับ cover image ที่ไม่จำเป็น
- ถ้า cover ไม่มี — แค่บอกใน metadata footer ว่า `📎 Cover image: —` ก็พอ ไม่ต้องเตือนเพิ่ม

## Multi-platform Variant

ถ้าผู้ใช้ระบุ "ทำทั้ง FB และ IG":
- generate 2 ไฟล์ใน folder เดียวกัน:
  - `final/<slug>/<N>-<topic>/<N>-<topic>-fb.md`
  - `final/<slug>/<N>-<topic>/<N>-<topic>-ig.md`
- เลข `<N>` ใช้ตัวเดียวกัน (เป็นโพสต์เดียวกัน คนละ platform)
- format ต่าง:
  - FB: ≤5 hashtag, line break ทุก 2-3 บรรทัด
  - IG: 8-15 hashtag block + first line ≤125 chars + เว้น 2 บรรทัดก่อน hashtag

## Validation

ก่อน save ตรวจ:
- [ ] ไม่มี frontmatter เหลืออยู่
- [ ] ไม่มี URL ใน body หลัก (เฉพาะ FB)
- [ ] Hashtag count ตรง platform sweet spot
- [ ] No trailing whitespace
- [ ] File + folder numbering ถูกต้อง (ไม่ซ้ำ)

## Re-format (revise)

ถ้าผู้ใช้บอก "แก้ post X ใหม่":
- โหลด `final/<slug>/<N>-<topic>/<N>-<topic>.md`
- ถามว่าจะแก้อะไร (เลข? topic slug? content?)
- ★ **ห้ามเปลี่ยน number** ของ folder เก่า — ถ้าจะแก้ใหม่จริงๆ ใช้ number เดิมทับ
- ถ้าผู้ใช้อยาก archive — rename folder เป็น `<N>-<topic>.archived/` แล้วใช้ number เดิมสำหรับเวอร์ชันใหม่
