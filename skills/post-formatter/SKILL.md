# Post Formatter

## Purpose

แปลง draft ที่ user แก้เสร็จแล้ว → final file ที่:
- พร้อม **copy paste** ลงเพจทันที (FB, IG, อื่นๆ)
- ตั้งชื่อด้วย **เลขลำดับ** (1, 2, 3...) ต่อเพจ — เห็น progression ของแต่ละเพจชัด
- มี **metadata footer** สำหรับ user (link สำหรับ first comment, alt-text, image path) — ส่วนนี้ไม่ paste ไปด้วย

## Folder Convention

```
<workspace>/
├── drafts/<slug>/
│   └── 2026-05-11-ai-misconception.md   ← raw (มี frontmatter)
└── final/<slug>/                          ← ★ output ของ skill นี้
    ├── 1-ai-misconception.md
    ├── 2-prompt-tips.md
    └── 3-...md
```

**Per-slug numbering** — แต่ละเพจมี sequence ของตัวเอง เริ่ม 1 ใหม่

## Preflight

1. **draft**: `drafts/<slug>/<file>.md`
   - ผู้ใช้ระบุไฟล์ → ใช้ตัวนั้น
   - ไม่ระบุ → ใช้ draft ใหม่สุดของ slug
   - ไม่มี draft → "ยังไม่มี draft รัน content-writer ก่อน" หยุด

2. **profile**: `business-profiles/<slug>/profile.yaml`
   - อ่าน `hashtags.always`, `hashtags.rotating`, `cta.links`
   - business_type (กำหนด tone ของ alt-text + format)

3. **ตรวจ folder `final/<slug>/`**:
   - ไม่มี → สร้าง
   - มีอยู่แล้ว → scan ไฟล์ใน folder หา max number ปัจจุบัน

## Auto-Numbering Logic

```python
# Pseudo-code
import re, glob
files = glob.glob(f"final/{slug}/*.md")
numbers = []
for f in files:
    name = os.path.basename(f)  # "1-topic.md"
    match = re.match(r"^(\d+)-", name)
    if match:
        numbers.append(int(match.group(1)))
next_n = max(numbers) + 1 if numbers else 1
```

ใช้ bash equivalent:
```bash
SLUG_DIR="final/<slug>"
mkdir -p "$SLUG_DIR"
MAX_N=$(ls "$SLUG_DIR" 2>/dev/null | grep -oE '^[0-9]+' | sort -n | tail -1)
NEXT_N=$((${MAX_N:-0} + 1))
```

## Topic Slug Generation

จาก draft frontmatter หา `hook_angle` หรือ topic title:

1. ถ้า draft มี `topic:` ใน frontmatter → ใช้ตัวนั้น
2. ไม่งั้น — ดึงจากชื่อไฟล์ draft: `2026-05-11-ai-misconception.md` → `ai-misconception`

แปลงเป็น kebab-case ภาษาอังกฤษ (หรือไทยก็ได้):
- ตัด stop words: "การ", "เรื่อง", "ที่"
- เก็บ keyword หลัก 3-5 คำ
- ใช้ `-` คั่น

ตัวอย่าง:
- `ai-misconception`
- `prompt-tips-for-marketer`
- `วิธี-เริ่มต้น-ai`

## Format Conversion

### 1. ตัด frontmatter ออก

draft มี:
```yaml
---
business: ai-edu
platform: facebook
date: 2026-05-11
hook_angle: misconception reveal
...
---
```

→ ตัดทิ้งทั้ง block

### 2. Process body

**Link routing** (FB):
- หา URL ทุกตัวใน body
- ตัดออก → เก็บไว้สำหรับ "first comment" section
- ถ้าใน body มีคำว่า "ลิงก์ใต้คอมเม้นต์" หรือ "link in bio" → คงไว้

**Hashtag arrangement**:
- ลบ hashtag ทั้งหมดจากกลางบทความ (ถ้ามี)
- รวบกับ `profile.hashtags.always` + 1-2 จาก rotating ที่ตรง topic
- วางท้ายโพสต์ (เว้น 1 บรรทัด)
- FB: ≤5 hashtags; IG: 8-15

**Line breaks**:
- ตรวจให้มี blank line ระหว่างย่อหน้า
- ห้ามมี trailing whitespace

### 3. Generate alt-text

ถ้ามี `cover_image:` ใน frontmatter:
- ดูจาก `image_prompt` (ถ้ามี) → แปลเป็นไทย ≤125 chars
- ไม่ใช่ "ภาพของ..." (redundant)
- บอกสิ่งที่เห็น ไม่ใช่ topic

ถ้าไม่มี image → ข้าม

## Output Format

ไฟล์ `final/<slug>/<N>-<topic>.md`:

```markdown
<post body — clean text พร้อม paste>

<hashtags>

---

📎 Cover image: <path or "—">
🔗 Link for first comment: <url or "—">
✏️ Alt-text: <auto-generated or "—">

📊 Char count: <X> | Hashtags: <N>/5 (FB)
```

**ส่วน metadata footer** ใต้ `---`:
- ไม่ใช่ส่วนของโพสต์ที่จะ copy ลงเพจ
- ใช้ดูสำหรับ user ตอน paste — รู้ว่าต้องใส่อะไรเป็น first comment, alt-text ของรูปคืออะไร
- เตือน user ใน output ว่า "copy เนื้อหาเหนือ `---` เท่านั้น"

## Multi-platform Variant

ถ้าผู้ใช้ระบุ "ทำทั้ง FB และ IG":
- generate 2 ไฟล์ — `<N>-<topic>-fb.md` + `<N>-<topic>-ig.md`
- เลข `<N>` ใช้ตัวเดียวกัน (เป็นโพสต์เดียวกัน คนละ platform)
- format ต่าง:
  - FB: ≤5 hashtag, line break ทุก 2-3 บรรทัด
  - IG: 8-15 hashtag block + first line ≤125 chars + เว้น 2 บรรทัดก่อน hashtag

## Validation

ก่อน save ตรวจ:
- [ ] ไม่มี frontmatter เหลืออยู่
- [ ] ไม่มี URL ใน body หลัก (เฉพาะ FB)
- [ ] Hashtag count ตรง platform sweet spot
- [ ] No trailing whitespace
- [ ] File numbering ถูกต้อง (ไม่ซ้ำกับที่มี)

## Output Message

หลัง save:

```
✅ Ready to paste:
   final/<slug>/<N>-<topic>.md

📋 วิธีใช้:
   1. เปิดไฟล์ ดูส่วนเหนือ "---" → copy ทั้งหมด paste ลงเพจ
   2. ส่วนใต้ "---" คือ metadata:
      - 📎 cover image: เอาขึ้นเพจตอนสร้างโพสต์
      - 🔗 link: paste เป็นคอมเมนต์แรก
      - ✏️ alt-text: ใส่ในช่อง alt ของรูป

📊 stats: <char count> chars | <hashtag count> hashtags
```

## Re-format (revise)

ถ้าผู้ใช้บอก "แก้ post X ใหม่":
- โหลด `final/<slug>/<N>-<topic>.md`
- ถามว่าจะแก้อะไร (เลข? topic slug? content?)
- ★ **ห้ามเปลี่ยน number** ของไฟล์เก่า — ถ้าจะแก้ใหม่จริงๆ ใช้ number เดิมทับ
- ถ้าผู้ใช้อยาก archive — rename เป็น `<N>-<topic>.archived.md` แล้วใช้ number เดิมสำหรับเวอร์ชันใหม่
